## 1-Descrição do Projeto

Aplicativo que demonstra o tempo de espera em cada especialidade de acordo com os processos médicos e laboratoriais em unidades de atendimento primário (UBS)

## 2-Business Drivers/Background

Tendo em vista o afogamento da logística hospitalar, pretendemos desenvolver um produto capaz de medir a fila de espera dos pacientes.

Outros produtos similares;

O [Agenda Fácil](https://play.google.com/store/apps/details?id=saude.prefeitura.sp.gov.br.autoagendamento&hl=pt_BR) é um aplicativo lançado pela Prefeitura de São Paulo e tem como objetivo de facilitar a vida do munícipe, ele verificar pedidos em fila de espera.

## 3-Visão

Facilitar o acompanhamento das filas de atendimento em clínicas médicas e unidades de saúde, visando aprimorar a logística do comparecimento do paciente

## 7-Riscos dos Projetos

**Riscos:**

* Defeitos do computador.	
* Afastamento de membros da equipe por doença.
* Desistência de membros da equipe.
* Mudanças de requisitos.
* Perda de backups

**Matriz de Risco**

| Probabilidade<br>Impacto | Baixo(20%) | Médio(>20%<60%) | Alto(>60%) |
| ------------------------ | :--------: | :-------------: | :--------: |
| Baixo(20%)               |            |      1,5        |            |
| Médio(>20%<60%)          |     3      |        4        |            |
| Alto(>60%)               |            |        2        |            |

**Estratégias mitigadoras**

| Risco | Grau        | Estratégia                                                   |
| ----- | ----------- | ------------------------------------------------------------ |
| 1     | Baixo-Médio | Disponibilizar computadores<br>de reserva para a equipe      |
| 2     | Médio-Alto  | Estarão disponíveis outros<br>colaboradores caso necessário. |
| 4     | Médio-Médio | Aplicar horas extras nas folgas.                             |
| 5     | Baixo-Médio | Desenvolver backup na Nuvem.                                 |
